# 安装

首先要保证电脑上已经安装了node，如果没有安装可以查看node windows安装及环境配置，Mac自行百度XD。请确保你的 Node.js 版本 >= 8。

# 安装vuepress包

yarn add -D vuepress@next # 或者：npm install -D vuepress@next

# 基本配置

# 默认的页面路由

站内的每一个文件夹下的 README.md 或者 index.md 文件都会被自动编译为 index.html，对应的链接将被视为 /，docs文件夹也就相当于路由的根目录，所有的路由查找都是基于它的。

文件的相对路径 页面路由地址
/README.md /
/guide/README.md /guide/
/config.md /config.html

# 导航栏链接设置

默认情况下导航栏只有一个搜索框以及title，打开.vuepress文件下的config.js，可以通过 themeConfig.nav 增加一些导航栏链接:

// .vuepress/config.js
module.exports = {
  themeConfig: {
    nav: [
      { text: 'Home', link: '/' },
      { text: 'Guide', link: '/guide/' },
      { text: 'External', link: 'https://google.com' },
    ]
  }
}

# 侧边栏配置

可以通过配置 themeConfig.sidebar 来设置侧边栏，它需要一个包含了多个链接的数组：

// .vuepress/config.js
module.exports = {
  themeConfig: {
    sidebar: [
      '/',
      '/page-a',
      ['/page-b', 'Explicit link text']
    ]
  }
}

# 侧边栏分组

如果想要侧边栏的项可以分组，也就是可以展开，需要使用对象，而不是上面例子中的字符串或者数组。

// .vuepress/config.js
module.exports = {
  themeConfig: {
    sidebar: [
      {
        title: 'Group 1',   // 必要的
        path: '/foo/',      // 可选的, 应该是一个绝对路径
        collapsable: false, // 可选的, 默认值是 true,
        sidebarDepth: 1,    // 可选的, 默认值是 1，也就是只会捕获一级标题
        children: [
          '/'
        ]
      },
      {
        title: 'Group 2',
        children: [ /* ... */ ]
      }
    ]
  }
}

# 多个侧边栏

如果你有多个页面，展示不同的信息，比如：前端内容一个页面，后端内容一个页面，这个时候你想要不同页面展示不同的侧边栏，需要将你的目录组织成下面的结构

.
├─ README.md
├─ contact.md
├─ about.md
├─ foo/
│  ├─ README.md
│  ├─ one.md
│  └─ two.md
└─ bar/
   ├─ README.md
   ├─ three.md
   └─ four.md

/ .vuepress/config.js
module.exports = {
  themeConfig: {
    sidebar: {
      '/foo/': [
        '',     /* /foo/ */
        'one',  /* /foo/one.html */
        'two'   /* /foo/two.html */
      ],

      '/bar/': [
        '',      /* /bar/ */
        'three', /* /bar/three.html */
        'four'   /* /bar/four.html */
      ],

      // fallback
      '/': [
        '',        /* / */
        'contact', /* /contact.html */
        'about'    /* /about.html */
      ]
    }
  }
}

# 搜索框

你可以通过设置 themeConfig.search: false 来禁用默认的搜索框，或是通过 themeConfig.searchMaxSuggestions 来调整默认搜索框显示的搜索结果数量：

module.exports = {
  themeConfig: {
    search: false,
    searchMaxSuggestions: 10
  }
}

# 最后更新时间

通过 themeConfig.lastUpdated 选项来获取每个文件最后一次 git 提交的 UNIX 时间戳(ms)，同时它将以合适的日期格式显示在每一页的底部：

module.exports = {
  themeConfig: {
    lastUpdated: 'Last Updated', // string | boolean
  }
}

# 公共文件

有时，你可能需要提供一个静态资源，但是它们并不直接被你的任何一个 markdown 文件或者主题组件引用 —— 举例来说，favicons 和 PWA 的图标，在这种情形下，你可以将它们放在 .vuepress/public 中， 它们最终会被复制到生成的静态文件夹中。

# 部署Github Pages

  1. 首先在github上新建立一个仓库，名称格式：用户名.github.io，新建时Owner下显示的即为你的用户名。
  2. 在你的项目中，创建一个如下的 deploy.sh 文件（请自行判断去掉高亮行的注释）:
  3. 使用命令行工具运行即可，成功后去github上的远程仓库查看是否文件已更新，然后访问 用户名.github.io 这个时候应该可以看到部署后的页面了。
#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run docs:build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages

cd -

# 部署配合Github Action

  1. 首先在存放 docs/.vuepress/dist 的仓库，添加一个Secret key，用于 Action 使用。
  2. 在该仓库点击 Aciton 操作，新建 workflow 文件
  3. 之后 push 操作时，就会自动触发部署流程
name: Blog Deploy Workflow
on:
  push: # 触发该 workflow 的操作以及分支，这里是master 发生 push 操作就会执行
    branches: [ master ]
jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout 🛎️
        uses: actions/checkout@master # If you're using actions/checkout@v2 you must set persist-credentials to false in most cases for the deployment to work correctly.
        with:
          fetch-depth: 2

      - name: Use Node.js 12.x  
        uses: actions/setup-node@v1
        with:
          node-version: '12.x'
  
      - name: Install and Build 🔧 # This example project is built using npm and outputs the result to the 'build' folder. Replace with the commands required to build your project, or remove this step entirely if your site is pre-built.
        run: |
          npm install
          npm run docs:build
          # Runs a single command using the runners shell

      - name: List output files
        run: ls -a docs/.vuepress/dist # 显示生成的目录文件

      - name: Deploy 🚀
        uses: peaceiris/actions-gh-pages@v3
        with:
          personal_token: ${{ secrets.ACCESS_TOKEN }} # 自己申请的密钥
          external_repository: xxxx/xxxx.github.io # 当然也可以利用gh-pages分支
          publish_branch: master # 部署到哪个分支
          publish_dir: ./docs/.vuepress/dist # 要部署的文件夹
